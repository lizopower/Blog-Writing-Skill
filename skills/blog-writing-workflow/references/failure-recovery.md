# Blog Writing Workflow Failure Recovery

## Audience Research Fails

Report limited audience evidence. Continue without pain point guidance only if the user accepts that limitation.

## Research Returns Minimal Data

Stop and ask whether to:
- provide more source material,
- refine the topic,
- proceed with limited evidence and explicit risk notes.

Do not invent missing evidence.

## File Parsing Fails

Report the specific file and likely reason. Ask for a better source format or continue with research-only evidence if acceptable.

## Data Validation Fails

Show validation issues. Do not continue to outline unless the user explicitly accepts the risk or the Context Pack is fixed.

## Strategy Pressure Test Finds Gaps

Return to `tech-research`, `audience-pain-point-research`, or `tech-blog-orchestrator`. Update `strategy.md` with unresolved risks.

## Visualization Data Is Missing

Skip visualization and mark charts manifest as empty. Do not invent values for charts.

## Fact Check Fails

Present the draft with issues flagged, or return to `tech-blog-writer` after correcting unsupported claims in the source artifacts.

Critical factual issues must be fixed before calling the article publication-ready.
