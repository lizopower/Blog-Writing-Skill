# Tech File Parser Skill - README

## 概述

**Tech File Parser** 是一个**通用的技术/B2B领域**文件解析 Skill。它从 PDF/Word/Excel 文件中提取结构化数据，输出为 JSON 格式，供后续图表生成和文章撰写使用。

## 角色定位

- **角色**: 资料解析员
- **职责**: 从文件中提取表格、时间序列、对比数据
- **输出**: 结构化 JSON（不写文章正文）
- **特色**: 来源定位 + 可视化建议

## 核心能力

### 1. 文档摘要
- 每个文件的主题、结论、适用范围
- 文件类型、创建日期、来源信息

### 2. 数据提取
- **表格**: 列名、单位、数据值、缺失值说明
- **时间序列**: 时间戳、测量值、趋势
- **对比数据**: 不同方案/产品/温度点/测试条件

### 3. 来源归属
- **PDF**: 页码、表号、章节
- **Word**: 标题层级、段落号、章节标题
- **Excel**: Sheet 名、单元格范围（如 "Sheet2!A1:D20"）

### 4. 可视化建议
- 识别适合可视化的数据
- 推荐图表类型（柱状图、折线图、饼图、对比表等）
- 说明为什么这个可视化有帮助

### 5. 结构化输出
- JSON 格式，便于脚本和工具解析
- 所有数据带来源标注
- 可直接用于图表生成和写作

## 不做什么

- ❌ 不写文章正文或叙述性文本
- ❌ 不对文件内容进行推断或假设
- ❌ 不生成图表/可视化（仅推荐）
- ❌ 不做 SEO 优化
- ❌ 不翻译或修改数据

## 输入输出

### 输入
```
files: 一个或多个文件（PDF/Word/Excel）
topic: 可选 - 博客主题上下文
```

### 输出
```json
{
  "files_processed": [...],      // 文件摘要
  "extracted_tables": [...],     // 提取的表格
  "time_series": [...],          // 时间序列数据
  "comparisons": [...],          // 对比数据
  "visualization_recommendations": [...], // 可视化建议
  "metadata": {...},             // 统计信息
  "quality_notes": {...}         // 质量提示
}
```

## 数据提取指南

### PDF 文件
- 提取表格并标注页码
- 解析图表（描述数据，不是图像）
- 从文本中提取测试条件
- 记录章节标题

### Word 文件
- 解析表格并保留章节上下文
- 从项目符号/列表中提取数据
- 映射标题层次结构
- 保留格式线索（加粗 = 重要）

### Excel 文件
- 每个 Sheet 单独处理
- 保留公式（描述计算）
- 记录单元格注释/备注
- 识别数据范围 vs. 计算

## 质量控制

### 输出前检查
- [ ] 所有表格有来源位置（页/Sheet/章节）
- [ ] 所有数值有单位
- [ ] 测试条件已提取（如有）
- [ ] 不确定数据已标记
- [ ] 提供了可视化建议
- [ ] JSON 有效且可解析
- [ ] 输出中无文章文本
- [ ] 无超出文件内容的推断

### 置信度级别
- **high**: 清晰的表格/图表，有单位和条件
- **medium**: 数据存在但上下文部分不清
- **low**: 模糊或需要解释
- **needs_confirmation**: 使用前应验证

## 特殊情况

### 扫描的 PDF
- 注明 OCR 可能有错误
- 标记低置信度的文本提取
- 如文本不可提取，描述图表/图像内容

### 复杂的 Excel 工作簿
- 每个 Sheet 单独解析
- 注明 Sheet 之间的关系
- 保留命名范围（如相关）
- 描述公式（不只是结果）

### 多语言文件
- 注明原始语言
- 无论如何提取数据（数字通用）
- 标记可能需要翻译的文本

### 文件中的图表/图形
- 描述显示的数据
- 如未提供表格，估算值
- 记录坐标轴标签和单位
- 如有源数据，建议重新创建

## 常见问题

**Q: 如果文件格式不支持怎么办？**  
A: 目前支持 PDF/Word/Excel。其他格式请转换后上传。

**Q: 扫描的 PDF 能解析吗？**  
A: 可以，但 OCR 可能有错误，会标注低置信度。

**Q: 能解析图片中的图表吗？**  
A: 可以描述图表内容，但精确数据需要表格形式。

**Q: 多语言文件能处理吗？**  
A: 可以提取数据（数字通用），文本会标记原始语言。

**Q: 输出的 JSON 如何使用？**  
A: 可用于自动生成图表、填充文章模板、数据验证等。
