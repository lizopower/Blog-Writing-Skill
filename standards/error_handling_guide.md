# 错误处理增强指南

## 版本
Version: 1.0.0
Last Updated: 2024-12-26

## 目的
为所有博客写作技能提供统一的错误处理机制，提升用户体验和系统稳定性。

## 错误分类

### 1. 用户输入错误 (User Input Error)
**严重性:** Low
**处理方式:** 友好提示 + 建议

**示例:**
```
❌ 错误的提示:
"Error: topic is undefined"

✅ 正确的提示:
"请提供文章主题。例如: '[产品/技术] 在 [极限条件部署场景] 中的应用'"
```

### 2. 数据质量错误 (Data Quality Error)
**严重性:** Medium
**处理方式:** 警告 + 继续执行（降级模式）

**示例:**
```
⚠️ 警告:
"检测到 3 个表格缺少单位信息。
建议: 手动补充单位后重新执行，或继续执行（可能影响图表生成）。
是否继续? [是/否]"
```

### 3. 系统错误 (System Error)
**严重性:** High
**处理方式:** 错误信息 + 重试建议

**示例:**
```
❌ 错误:
"文件解析失败: performance_test_report.pdf
原因: 文件损坏或格式不支持
建议:
1. 检查文件是否完整
2. 尝试重新导出 PDF
3. 联系支持团队"
```

## 错误处理模式

### 模式 1: 重试机制

```markdown
适用场景:
- 网络请求失败
- 临时文件访问失败
- API 调用超时

实现:
1. 最多重试 3 次
2. 每次重试间隔递增 (1s, 2s, 4s)
3. 记录重试次数和原因
4. 3 次后仍失败 → 返回错误
```

### 模式 2: 降级策略

```markdown
适用场景:
- 可选功能失败
- 数据不完整但可继续

示例:
- Research 失败 → 仅使用文件数据
- Visualization 失败 → 跳过图表，继续写作
- 部分表格缺少单位 → 标记警告，继续处理
```

### 模式 3: 检查点恢复

```markdown
适用场景:
- 长时间运行的流程
- 多步骤工作流

实现:
1. 每个步骤完成后保存状态
2. 失败时从最后检查点恢复
3. 避免重复执行已完成步骤
```

## 标准错误消息格式

```json
{
  "error": {
    "code": "ERROR_CODE",
    "severity": "low" | "medium" | "high",
    "message": "用户友好的错误描述",
    "details": "技术细节（可选）",
    "suggestions": [
      "建议 1",
      "建议 2"
    ],
    "retry_available": true,
    "context": {
      "skill": "tech-file-parser",
      "step": "parse_pdf",
      "file": "performance_test_report.pdf"
    }
  }
}
```
